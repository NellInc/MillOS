## Summary

MillOS has three test layers: Vitest unit tests (in `src/__tests__/`), Playwright E2E tests (in `e2e/`), and a coverage configuration. Playwright targets localhost:3000. The CI pipeline retries E2E tests on failure and serializes workers on CI. (playwright.config.ts; vitest.config.ts)

## Vitest Unit Tests

`src/__tests__/` contains integration tests. `src/stores/__tests__/` contains store-level unit tests. Vitest config is in `vitest.config.ts` at repo root; setup file is `vitest.setup.ts`.

Key test file observed: `src/__tests__/ai-command-center.spec.ts` — tests the AICommandCenter component which is the primary game control surface.

## Playwright E2E Tests

Configuration: `playwright.config.ts`. Settings:
- `testDir: './e2e'`
- `baseURL: 'http://localhost:3000'`
- `fullyParallel: true`
- CI: `retries: 2`, `workers: 1` (serialized)
- Dev: `retries: 0`, workers unconstrained
- Trace on first retry; screenshot on failure

E2E test directory: `e2e/` (one file observed: `ai-command-center.spec.ts`).

## Coverage

`coverage/` at repo root — generated coverage reports. Coverage config is in `vitest.config.ts`.

## Performance Testing

Dedicated perf test scripts at repo root:
- `perf-test.cjs`, `perf-test-gpu.cjs`, `perf-test-devtools.cjs`
- Output files: `perf-test-output.txt`, `perf-test-output-after-fix.txt`, `perf-test-gpu-output*.txt`

These appear to be standalone Node scripts for measuring render performance, GPU memory, and DevTools profiling. Not part of the Vitest/Playwright suite.

## Store Tests

`src/stores/__tests__/` — individual store tests. The complex stores (multiplayerStore, workerPersonalityStore, etc.) have dedicated test files.

## Playwright Report

`playwright-report/` and `test-results/` at repo root — generated by Playwright on test runs.

## Make Targets

From CLAUDE.md: `make validate-lean` (~30s iteration), `make lint && make typecheck` (~15s quick), `make validate-all` (~3min final). These are the canonical validation commands.
